## 🎯 Switch-Code란?

### 간단하게 설명하면...

**Switch-Code**는 AI 개발 도구(Claude Code, Codex)에서 사용하는 **AI Provider를 쉽게 바꿀 수 있는 앱**입니다.

```
예시:
오전에는 OpenAI로 작업 → 오후에는 DeepSeek으로 전환
비용 절감을 위해 Anthropic → OpenRouter로 변경
```

### 왜 필요한가요?

Claude Code나 Codex를 사용할 때:
- ❌ **설정 파일을 직접 수정하는 건 번거롭고 실수하기 쉬움**
- ❌ **여러 Provider를 사용하려면 매번 설정 파일 편집 필요**
- ❌ **API Key 관리가 복잡함**

Switch-Code를 사용하면:
- ✅ **클릭 한 번으로 Provider 전환**
- ✅ **API Key와 설정을 안전하게 관리**
- ✅ **여러 Provider를 미리 등록해두고 상황에 맞게 선택**

## 📦 설치하기

### 방법 1: 간편 설치 (Mac 권장) 🍎

Mac 사용자라면 Homebrew로 쉽게 설치할 수 있습니다:

```bash
# 1. Homebrew로 필수 도구 설치
brew install pnpm rust

# 2. Switch-Code 다운로드
git clone https://github.com/reallygood83/switch-model.git
cd switch-model

# 3. 의존성 설치
pnpm install

# 4. 실행!
pnpm dev
```

### 방법 2: 직접 빌드 🔧

#### 사전 준비물
- **Node.js 18** 이상 ([다운로드](https://nodejs.org/))
- **Rust 1.85** 이상 ([다운로드](https://www.rust-lang.org/))
- **pnpm 8** 이상

#### 설치 단계

```bash
# 1. pnpm 설치 (아직 없다면)
npm install -g pnpm

# 2. Switch-Code 다운로드
git clone https://github.com/reallygood83/switch-model.git
cd switch-model

# 3. 의존성 설치
pnpm install

# 4. 개발 모드로 실행
pnpm dev
```

### 방법 3: 배포용 앱 만들기 📱

설치형 앱을 만들려면:

```bash
# 빌드 명령 실행
pnpm build

# 빌드된 앱 위치:
# - macOS: src-tauri/target/release/bundle/dmg/
# - Windows: src-tauri/target/release/bundle/msi/
# - Linux: src-tauri/target/release/bundle/appimage/
```

## 🎬 첫 실행 및 설정

### 1단계: 앱 실행

```bash
cd switch-model
pnpm dev
```

### 2단계: 튜토리얼 따라하기

앱을 처음 실행하면 **4단계 튜토리얼**이 자동으로 시작됩니다:

#### 📚 튜토리얼 내용

1. **Provider란 무엇인가?**
   - AI 서비스 제공 회사 (OpenAI, Anthropic, DeepSeek 등)
   - TV 채널을 바꾸듯 Provider를 전환하는 개념

2. **API Key 추가하기**
   - API Key는 "출입증"처럼 인증을 위한 비밀번호
   - 각 Provider 웹사이트에서 발급받아서 등록

3. **Provider 전환하기**
   - "전환" 버튼 클릭으로 즉시 변경
   - 터미널 재시작 후 적용

4. **터미널 재시작하기**
   - Claude Code나 Codex 터미널을 종료했다가 다시 시작
   - 새로운 Provider 설정 적용 완료!

### 3단계: 언어 설정 (선택사항)

우측 상단 언어 선택기에서:
- 🇰🇷 **한국어**
- 🇺🇸 English
- 🇨🇳 中文

## ➕ Provider 추가하기

### 예시: DeepSeek Provider 추가

#### 1️⃣ "공급자 추가" 버튼 클릭

메인 화면 우측 상단의 **"+ 공급자 추가"** 버튼을 클릭합니다.

#### 2️⃣ 정보 입력

| 필드 | 입력 내용 | 설명 |
|------|----------|------|
| **공급자 이름** | `DeepSeek` | 원하는 이름으로 설정 (예: DeepSeek Pro, OpenRouter 등) |
| **웹사이트 URL** | `https://platform.deepseek.com` | Provider의 공식 웹사이트 주소 |
| **API 엔드포인트** | `https://api.deepseek.com` | API 호출 주소 (프리셋 자동 완성) |
| **API Key** | `sk-xxxxxxxxxxxxx` | DeepSeek 웹사이트에서 발급받은 키 |

#### 3️⃣ 프리셋 사용 (권장)

**"프리셋 선택"** 버튼을 클릭하여:
- DeepSeek
- OpenAI
- Anthropic
- OpenRouter
- Azure
- Google

등 주요 Provider의 설정을 자동 완성할 수 있습니다.

#### 4️⃣ 저장 및 확인

- **"저장"** 버튼 클릭
- 메인 화면에 새 Provider 카드가 추가된 것 확인

## 🔄 Provider 전환하기

### 기본 전환 방법

1. **전환할 Provider 카드 찾기**
2. **"전환" 버튼 클릭**
3. **터미널 재시작**

```bash
# Claude Code 사용 중이라면:
# VS Code 터미널 종료 → 새 터미널 열기

# Codex 사용 중이라면:
# Codex 앱 재시작
```

### 전환 확인 방법

터미널에서 다음 명령으로 현재 Provider 확인:

```bash
# Claude Code
cat ~/.config/claude-code/config.json | grep baseURL

# Codex
cat ~/.codex/config.json | grep baseURL
```

## 🔧 고급 기능

### 1️⃣ MCP (Model Context Protocol) 관리

**MCP란?** Claude Code와 Codex에서 사용하는 확장 기능 시스템입니다.

#### MCP 서버 추가하기

1. **"MCP 관리"** 탭 클릭
2. **"+ MCP 추가"** 버튼
3. **정보 입력**:
   - MCP 제목 (고유 식별자)
   - 표시 이름
   - JSON 또는 TOML 설정
4. **"저장"** 클릭

#### 프리셋 MCP 서버

Switch-Code는 다음 MCP 서버를 프리셋으로 제공:
- **mcp-server-fetch**: HTTP 요청 도구
- **time-server**: 시간 쿼리 도구
- **memory-server**: 메모리 관리
- **sequential-thinking**: 체계적 사고 지원
- **context7**: 문서 검색 도구

### 2️⃣ 설정 가져오기/내보내기

#### 내보내기 (백업)

1. **"설정"** 탭 이동
2. **"파일로 설정 내보내기"** 클릭
3. JSON 파일 저장

#### 가져오기 (복원)

1. **"설정"** 탭 이동
2. **"가져오기"** 버튼
3. 백업한 JSON 파일 선택

### 3️⃣ Provider 복제

기존 Provider 설정을 복사하여 새로운 Provider 생성:

1. Provider 카드에서 **"복제"** 버튼 (📋 아이콘)
2. 이름 수정 후 저장

### 4️⃣ 속도 테스트

API 엔드포인트의 응답 속도 측정:

1. Provider 편집 화면에서 **"속도 테스트"** 클릭
2. 평균 응답 시간 확인 (ms 단위)

### 5️⃣ 수동 정렬

Provider 카드를 드래그 앤 드롭으로 순서 변경:
- 자주 사용하는 Provider를 상단에 배치
- 프로젝트별로 그룹화

## ❓ 문제 해결

### Q1: Provider 전환 후 적용이 안 돼요

**해결 방법**:
1. **터미널 완전 종료** 후 재시작
2. VS Code 완전 종료 후 재실행
3. Switch-Code에서 **"Claude 플러그인에 적용"** 옵션 활성화 확인

### Q2: API Key 오류가 나요

**확인 사항**:
- API Key가 정확한지 재확인
- API Key 앞뒤 공백 제거
- Provider 웹사이트에서 Key 만료 확인
- API 크레딧 잔액 확인

### Q3: 앱이 실행되지 않아요

**해결 방법**:
```bash
# 의존성 재설치
pnpm install

# 캐시 클리어
pnpm store prune

# 다시 실행
pnpm dev
```

### Q4: 설정 파일 위치를 모르겠어요

**기본 경로**:
```bash
# Claude Code
~/.config/claude-code/config.json

# Codex
~/.codex/config.json

# Switch-Code 설정
~/.switch-code/settings.json
```

### Q5: WSL 환경에서 사용하고 싶어요

**설정 방법**:
1. **"설정"** → **"고급"** 탭
2. **"설정 디렉토리 재정의"** 섹션
3. WSL 내부 경로 지정:
   ```
   Claude: \\wsl$\Ubuntu\home\user\.config\claude-code
   Codex: \\wsl$\Ubuntu\home\user\.codex
   ```
4. 저장 후 앱 재시작

## 💡 추가 팁

### Provider별 특징 비교

| Provider | 장점 | 단점 | 추천 용도 |
|----------|------|------|----------|
| **OpenAI** | 강력한 성능, 풍부한 문서 | 비용 높음 | 프로덕션 코드 |
| **Anthropic** | Claude 3.5 Sonnet, 긴 컨텍스트 | 미국 외 지역 제한 | 복잡한 분석 |
| **DeepSeek** | 저렴한 가격, 빠른 응답 | 영어 외 성능 낮음 | 일상 작업 |
| **OpenRouter** | 다양한 모델 선택, 유연성 | 설정 복잡 | 실험 및 비교 |

### API Key 보안 관리

- ✅ `.env` 파일에 저장하고 `.gitignore`에 추가
- ✅ 정기적으로 Key 로테이션
- ✅ 사용하지 않는 Key는 즉시 삭제
- ❌ 절대 GitHub에 업로드 금지
- ❌ 팀원과 공유하지 말 것

### 비용 절감 팁

1. **상황별 Provider 선택**:
   - 간단한 작업: DeepSeek, OpenRouter
   - 중요한 작업: OpenAI, Anthropic

2. **MCP 활용**:
   - 반복 작업은 MCP 서버로 자동화
   - 비용 효율적인 워크플로우 구축

3. **토큰 사용량 모니터링**:
   - Provider 대시보드에서 사용량 확인
   - 예산 알림 설정
